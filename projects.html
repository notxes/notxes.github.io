Enterprise Segmented Cybersecurity Homelab

Overview

Designed and built a segmented cybersecurity homelab to simulate a realistic enterprise environment. The lab supports both attacker and defender workflows, allowing me to practice network security, monitoring, incident response, and adversary simulation in a controlled setting.

Technologies Used

  • Proxmox
  • OPNsense
  • VLAN Segmentation
  • Security Onion
  • Active Directory
  • Kali Linux
  • Pi-hole
  • WireGuard
  • Cisco Switching

Architecture and Design

  • VLAN 10 → Management
  • VLAN 20 → Red Team / Attacker Systems
  • VLAN 21 → Blue Team / Victim Systems
  • VLAN 30 → SOC / Monitoring Infrastructure
  • VLAN 40 → DMZ / Public-Facing Services
  • VLAN 50 → IoT / Untrusted Devices
  • Firewall rules configured in OPNsense to control inter-VLAN traffic
  • Mirrored traffic sent to Security Onion for monitoring and alerting
  • Virtual networking built with Proxmox bridges and VLAN tagging

What I Did

  • Built the virtualized infrastructure across multiple Proxmox systems
  • Designed and implemented segmented networks for realistic attack and defense scenarios
  • Configured DMZ and IoT network segmentation to strengthen isolation and trust boundaries
  • Set up attacker and victim systems for testing and monitoring
  • Integrated Security Onion for traffic visibility, alert review, and analysis
  • Configured DNS, VPN access, and supporting infrastructure for lab functionality
  • Applied firewall rules to restrict lateral movement between VLANs while allowing controlled access where needed
  • Tested simulated attack activity to better understand detection and response workflows

Results and Learning Outcomes

This project strengthened my understanding of network segmentation, monitoring visibility, traffic flow, firewall policy design, and incident response. Building dedicated management, attacker, victim, SOC, DMZ, and IoT segments gave me practical experience with trust boundaries and how proper isolation can reduce attack surface and limit lateral movement inside an environment.

Security Onion Monitoring Environment

Overview

Deployed a Security Onion monitoring environment to observe network traffic across lab segments and better understand SOC workflows such as alerting, traffic analysis, event investigation, and visibility tuning.

Technologies Used

  • Security Onion
  • Zeek
  • Suricata
  • Proxmox
  • Cisco Switch SPAN Port
  • Ubuntu Server

Architecture and Design

  • Configured separate management and monitoring interfaces
  • Mirrored traffic from selected VLANs to the monitoring interface
  • Used a SPAN port to feed lab traffic into Security Onion
  • Monitored segmented activity to improve visibility into attacker and victim communications

What I Did

  • Installed and configured Security Onion in a standalone lab deployment
  • Set up network monitoring for segmented lab traffic
  • Verified packet capture and log ingestion
  • Reviewed dashboards, alerts, and SOC-style workflows
  • Used simulated attack traffic to understand what activity was visible and how alerts were generated
  • Troubleshot interface, span, and traffic visibility issues during deployment

Results and Learning Outcomes

This project helped me better understand how network monitoring tools ingest traffic, generate alerts, and support investigation workflows. It also improved my troubleshooting skills around mirrored traffic, interface configuration, and visibility across segmented environments.

What I Did

  • Focused on the cybersecurity and protection side of the design

  • Supported the overall technical and business design of the environment
